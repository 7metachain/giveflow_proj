
目标：在 Monad Hackathon 中打造一个“零抽成、全透明、AI 驱动”的慈善捐助平台，
让每一笔善款都可追踪、可解释、可验证，且支持小额高频捐赠。

---
迭代版本：V2（以创新性 + 区块链必要性 + Monad 特性为优先）

设计原则：
- Web3 不是噱头：每个链上动作都必须带来信任、效率或成本优势。
- 创新可演示：2 分钟内让评委一眼看到“为什么必须在链上”。
- Monad 原生能力可见：并行执行、低 Gas、高吞吐被具象为功能与体验。

V2 一句话 Pitch：
GiveFlow 是一个基于 Monad 的可验证慈善资金网络，依靠链上“可证性 + 自动释放 + 并行微捐”
让资金透明到每一笔、捐赠精细到每一分钱、并用 AI 解释每一次资金流转。

V2 迭代文档

1. 目标与衡量标准（确保“不是为了 Web3 而 Web3”）
目标：用链上可证性和 Monad 的成本/性能优势解决“信任、成本、效率”三大问题。
衡量标准：
- 透明度：每笔捐赠可在链上追溯到具体项目与里程碑释放记录。
- 成本：支持 $0.01 起捐 + 每周自动微捐（单笔 Gas 成本低于 $0.01 等值）。
- 体验：一次交互支持多项目捐赠（batchDonate）与实时进度更新（捐赠流）。

2. 创新点（与传统 Web3 慈善明确拉开差距）
2.1 可验证慈善资金流（Verifiable Flow）
- 捐赠-里程碑-提款三段链上记录，形成可验证资金轨迹。
- 里程碑释放为“程序化透明”，而不是平台口头承诺。
2.2 并行微捐网络（Parallel Micro-Donations）
- 支持一次交易向多个项目捐赠（批量微捐），体现 Monad 并行执行。
- AI 解析“捐 3 个项目各 $1” → 同步构建并行交易。
2.3 可解释影响力（Explainable Impact）
- AI 结合链上事件生成“可解释的资金去向”与影响力叙述。
- 输出可分享的影响力报告，包含链上证明链接。

3. 为什么必须上链（链上必要性论证）
- 信任最小化：捐款、释放、提款不可篡改，避免平台黑箱。
- 资金安全：里程碑未达成时无法提现（合约约束）。
- 全球协作：跨境捐赠无需信任中介，链上结算即时可查。
- 公开审计：任何人可验证资金去向，形成“公共账本的社会信用”。

4. Monad 特性映射（功能级体现）
4.1 并行执行 → 批量微捐 / 多项目一键捐
- feature：batchDonate + 多项目并行执行
- demo：一次操作向 5 个项目捐 5 笔 $0.5
4.2 低 Gas → 高可持续微订阅
- feature：weekly/monthly 低成本订阅
- demo：每周 $1 捐赠自动执行，Gas 成本可忽略
4.3 高吞吐 & 低延迟 → 实时资金流可视化
- feature：实时捐赠流与里程碑更新
- demo：页面实时刷新“第 X 笔捐款 / 第 X 次里程碑释放”

5. 核心链上设计（精简但完整的闭环）
5.1 合约结构（MVP）
- CampaignRegistry：创建项目、基本参数（标题、目标金额、截止日期、受益地址）
- MilestoneVault：里程碑与分阶段释放（链上记录 + 释放条件）
- BatchDonate：并行批量捐赠入口
- RecurringDonation：低成本定期捐赠
5.2 关键事件（前端可视化所需）
- DonationMade(projectId, donor, amount, token)
- MilestoneSet(projectId, milestoneId, targetAmount, hash)
- MilestoneReleased(projectId, milestoneId, amount)

6. MVP 范围（2-3 分钟可演示闭环）
6.1 必做
- 项目列表 → 选择项目 → 一键捐赠（链上成功）
- 资金流可视化（Donation → Milestone → Release）
- 批量微捐（一次捐多个项目）
6.2 可选加分
- 定期捐赠订阅
- 影响力报告自动生成（含链上链接）

7. Demo 流程（面向评委的“必上链”叙事）
1) “为什么必须上链”：展示透明可查的捐赠与里程碑记录
2) “Monad 为什么关键”：展示 batchDonate + 低 Gas 微捐
3) “AI 的价值”：将链上数据解释成可理解的影响力报告

8. 风险与规避（评委关心点提前回应）
- 风险：项目真实性 → 方案：里程碑证明哈希上链 + 多签审核
- 风险：用户门槛 → 方案：AI 意图驱动 + 简化签名步骤
- 风险：数据复杂 → 方案：统一事件日志 + 可视化模板

2. 核心架构功能

2.1 系统架构总览

用户(捐赠者/发起人/受益人)
        │
        ▼
前端(Next.js + shadcn/ui + Wallet)
        │
        ▼
API层(/api/chat, /api/campaigns, /api/donate, /api/track)
        │
        ├── AI层(OpenAI/Vercel AI SDK): 意图解析、推荐、解释
        ├── Web3层(wagmi/viem): 交易构建、签名执行
        └── 数据层(Supabase/DB): 项目元数据、里程碑证明、图片等
        │
        ▼
Monad 区块链
  - CharityCampaign 合约
  - DonationVault 合约
  - RecurringDonation 合约

2.2 核心功能模块

模块 A：募捐项目管理

- 创建募捐项目（标题、描述、目标金额、截止日期、受益地址）
- 设置里程碑（阶段目标与用途）
- 更新进展证明（链下存储 + 链上哈希）
- 资金分阶段释放（里程碑完成后释放）
模块 B：捐赠体验与资金流转

- 单次捐赠（Token 或稳定币）
- 批量微捐（多项目多笔并行）
- 定期捐赠订阅（每日/每周/每月自动捐）
- 资金直达（不经过平台抽成）
模块 C：透明追踪与影响力报告

- 资金流可视化（Donation -> Campaign -> Withdraw）
- 里程碑兑现记录
- 捐赠历史与汇总统计
- 自动生成影响力报告（可分享）
模块 D：AI 意图驱动体验

- 自然语言搜索项目
- 自然语言完成捐赠与设置订阅
- AI 解读资金去向与进展
- AI 生成项目推荐与个性化匹配

5. 更好体现 Monad 特性应侧重哪些？

5.1 Monad 的关键特性

- 高性能并行执行
- 极低 Gas 成本
- 高吞吐与低延迟确认
5.2 对应可突出功能（重点展示）

- 高频捐赠也能实时更新资金流进度
- 页面实时刷新显示“第 X 笔捐款”
重点 3：定期捐赠订阅（低成本持续交互）

- 每周自动扣款而不担心高 Gas
- 做成“微订阅”概念，体现 Monad 经济性
重点 4：AI + Monad 并行执行

- AI 解析多步骤请求时，可并行构建交易
- Demo 展示：AI 处理“捐 3 个项目各 $1” 一次完成
